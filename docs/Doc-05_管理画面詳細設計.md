# INK ADDICTION LP – 管理画面詳細設計 v1.0

受領日: 2026-02-27  
ステータス: 確定

---

## 1. 目的

本書は、INK ADDICTION LP の管理画面（Admin Console）の画面仕様・操作フロー・状態遷移・UI構成・誤操作防止設計を定義する。

本管理画面は単なるCMSではなく、以下を実現する運用基盤である。

- 作品の質管理
- 自動分類の確認と修正
- スクショ解析の承認制御
- 問い合わせの高速処理
- 誤公開の防止
- 将来的なTattooBase統合拡張

---

## 2. ロール設計

| ロール                | 権限                                                       |
| --------------------- | ---------------------------------------------------------- |
| **Admin（兄）**       | 全機能利用可能 / 公開操作 / スロット承認 / 設定変更        |
| **Staff（将来拡張）** | 問い合わせ閲覧・返信のみ / 作品編集不可 / スロット公開不可 |

## 4. 作品管理画面

### 4.1 `/admin/works`

**一覧表示項目**

| 項目         | 備考                       |
| ------------ | -------------------------- |
| サムネ       | 1枚目の作品画像            |
| タイトル     |                            |
| `finalGenre` | 未設定の場合「要確認」表示 |
| `autoGenre`  | AI推定結果                 |
| `confidence` | 数値 + 低い場合は警告色    |
| 公開状態     | 公開 / 非公開              |
| 作成日       |                            |

**フィルタ**

- 公開 / 非公開
- ジャンル
- confidence低のみ

**アクション**

- 編集
- 公開切替
- ソフト削除

---

### 4.2 `/admin/works/new`

**入力項目**

| 項目                     | 必須 |
| ------------------------ | ---- |
| タイトル                 | ✅   |
| 説明                     |      |
| 画像アップロード（複数） | ✅   |
| 初期ジャンル             |      |

**保存後の処理**

1. 分類ジョブ起動
2. `autoGenre` 保存

---

### 4.3 `/admin/works/{id}`

**表示・編集項目**

| 項目                                  | 操作                                       |
| ------------------------------------- | ------------------------------------------ |
| 画像一覧                              | 並び替え可能                               |
| `autoGenre` + `confidence`            | 参照のみ                                   |
| `finalGenre`                          | セレクトボックスで編集                     |
| タグ                                  | 編集可                                     |
| 価格目安（`price_min` / `price_max`） | 編集可                                     |
| セッション目安                        | 編集可                                     |
| 公開トグル                            | `finalGenre` 設定済み・画像1枚以上で有効化 |

**制御ルール**

> [!CAUTION]
>
> - `finalGenre` 未設定では公開不可（UIでトグル無効化）
> - 画像0枚では公開不可
> - 公開操作時は確認モーダルを必ず表示する

**監査ログ記録対象**

- ジャンル変更
- 公開状態変更

## 5. スクショ取り込み管理

### 5.1 `/admin/schedule-import`

**表示**

- スクショアップロードUI
- 過去履歴一覧
- 処理状態

**処理状態の表示**

| ステータス   | 表示                           |
| ------------ | ------------------------------ |
| `pending`    | 処理待ち                       |
| `processing` | 解析中                         |
| `success`    | 抽出完了                       |
| `failed`     | 失敗（手動入力導線を強調表示） |

---

### 5.2 `/admin/schedule-import/{id}`

**表示項目**

| 項目               | 内容                               |
| ------------------ | ---------------------------------- |
| 原本画像           | スクショのプレビュー               |
| 候補スロット一覧   | `startAt` / `endAt` / `confidence` |
| 編集可能フィールド | 時間の修正                         |
| 削除チェック       | スロット単位で除外可能             |
| 手動追加ボタン     | OCR未検出スロットを手動追加        |

**アクション**

| アクション    | 処理                                            |
| ------------- | ----------------------------------------------- |
| 保存（draft） | `availability_slots` に `status = draft` で保存 |
| 承認して公開  | `status = published` で insert + 監査ログ保存   |

> [!CAUTION]
> 承認ボタン押下時のみ `availability_slots` へ INSERT する。自動公開は実装禁止。

## 6. 空き枠管理画面

### 6.1 `/admin/availability`

**表示方法**

- カレンダービュー
- リストビュー（切替可）

**表示項目**

| 項目         | 備考                                              |
| ------------ | ------------------------------------------------- |
| 日時         | `start_at` / `end_at`                             |
| ステータス   | `published / hidden / booked / draft / tentative` |
| source       | `screenshot / manual`                             |
| `confidence` | スクショ由来の場合のみ                            |

**アクション**

| アクション | 処理                              |
| ---------- | --------------------------------- |
| 編集       | 時間修正（重複チェック実施）      |
| 非表示     | `status = hidden`                 |
| booked化   | `status = booked`（Public非表示） |
| 削除       | ソフト削除                        |

**制御**

- `booked` 化するとPublicから非表示になる
- 時間編集時に重複スロットの有無をチェックし、重複がある場合は保存を拒否する

## 11. UX配慮

| 状況                  | 対応                       |
| --------------------- | -------------------------- |
| `confidence` 低い作品 | 一覧で警告色・アイコン表示 |
| 公開直前              | 確認モーダル必須           |
| OCR失敗時             | 手動入力導線を強調表示     |
| 未対応問い合わせ      | 赤バッジ表示               |

---

## 12. エッジケース設計

| ケース               | 対応                                            |
| -------------------- | ----------------------------------------------- |
| OCR失敗              | `extraction_status = failed` 表示・手動入力導線 |
| 分類API失敗          | `autoGenre = null`・「要確認」表示              |
| 大量画像アップロード | キュー処理（Worker非同期）                      |
| 空き枠重複           | 保存拒否・エラーメッセージ表示                  |

---

## 13. パフォーマンス配慮

- 一覧はサーバーサイドページネーション
- 画像はサムネのみ表示（詳細画面で高解像を遅延ロード）
- 監査ログはデフォルトで最新100件を表示

## 15. 受け入れ基準

| 基準                    | 内容                     |
| ----------------------- | ------------------------ |
| 未承認データ            | Public に表示されない    |
| `finalGenre` 未設定作品 | 公開不可（UIで制御）     |
| `booked` スロット       | Public 非表示            |
| 問い合わせ添付          | Public アクセス不可      |
| 重要操作                | 全て監査ログに記録される |
