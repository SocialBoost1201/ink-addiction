# INK ADDICTION LP – UI デザインシステム v1.0

受領日: 2026-02-27  
ステータス: 確定

---

## 1. 目的

本書は、INK ADDICTION LP のUI設計基準を定義する。  
デザインの方向性は「Apple公式サイトのような洗練・余白・写真主役」を採用し、感情に訴えつつ、静かに信頼を構築する。

> **UIは派手さではなく、質感・間・構造で勝つ。**

---

## 2. デザイン原則

### 2.1 Core Principles

- 余白を恐れない
- 写真を主役にする
- 文字は少なく、強く
- CTAは最小限
- モーションは静かに
- 黒と白を基調、アクセント1色のみ

### 2.2 Emotional Tone

| トーン               | 意図                   |
| -------------------- | ---------------------- |
| クール               | 過剰な主張をしない     |
| 洗練                 | Apple水準の整理        |
| 信頼                 | 安全性・誠実さの可視化 |
| 高価格帯に見合う質感 | 安っぽさの排除         |

---

## 3. カラーデザイン

### 3.1 ベースカラー

| 役割             | カラーコード |
| ---------------- | ------------ |
| Background Light | `#FFFFFF`    |
| Background Dark  | `#0F0F0F`    |
| Primary Text     | `#111111`    |
| Secondary Text   | `#6B6B6B`    |

### 3.2 アクセントカラー（1色のみ）

```
#7A1E1E  ── 深い赤寄りワインカラー
```

**採用理由**

- タトゥーとの親和性
- 高級感
- 攻撃的すぎない

### 3.3 使用ルール

| 用途       | ルール               |
| ---------- | -------------------- |
| CTAボタン  | アクセントカラーのみ |
| リンク下線 | アクセントカラー     |
| hover      | 明度10%変化          |
| 多色展開   | 禁止                 |

---

## 4. タイポグラフィ

### 4.1 フォント

| 用途   | フォント                |
| ------ | ----------------------- |
| 日本語 | Noto Sans JP / Inter JP |
| 英数字 | Inter（または同系統）   |

### 4.2 タイプスケール

| サイズ | 用途                           |
| ------ | ------------------------------ |
| 12px   | Caption                        |
| 14px   | Small                          |
| 16px   | Body                           |
| 20px   | Subheading                     |
| 24px   | Section Heading                |
| 32px   | Large Heading                  |
| 40px   | Hero                           |
| 56px   | Large Hero（デスクトップのみ） |

### 4.3 ルール

- 1セクション最大2階層
- 長文は避ける
- 改行でリズムを作る

---

## 5. レイアウト設計

### 5.1 グリッド

| 設定               | 値                     |
| ------------------ | ---------------------- |
| 最大幅             | 1200px                 |
| 左右余白           | 24px（モバイル: 16px） |
| セクション上下余白 | 96px（モバイル: 64px） |

### 5.2 ヒーロー

- 全幅写真
- 中央または左揃えコピー
- CTA 1つのみ
- 背景に薄いグラデーション可（控えめに）

### 5.3 セクション構造

```
[Section Header]
[Visual / Content]
[Single CTA（optional）]
```

> **CTA乱立禁止**

---

## 6. コンポーネント設計

### 6.1 Buttons

| 種別          | 仕様                                                 |
| ------------- | ---------------------------------------------------- |
| **Primary**   | 背景: アクセント / 文字: 白 / 角丸: 8px / 高さ: 48px |
| **Secondary** | 背景: 透明 / 枠線: 1px / 文字: Primary Text          |
| **Disabled**  | opacity: 0.4                                         |

### 6.2 Work Card

- 縦長レイアウト
- 写真比率: **4:5 推奨**
- hover時: 2%拡大（transition付き）
- タイトルは最小表示

### 6.3 Filter Chips

- 枠線あり（通常時）
- 選択時: アクセント背景 + 白文字

### 6.4 Inquiry Form（マルチステップ）

- 1画面最大4項目
- 進捗バー表示
- 入力フォーカス時: アクセントカラーの枠

### 6.5 Upload Dropzone

- ドラッグ＆ドロップエリア
- アップロード後に画像プレビュー表示
- 削除ボタン付き
- 最大枚数の残数表示

### 6.6 Availability Calendar

- シンプルリスト優先
- 日付は大きめ表示
- 空き枠のみ表示
- `booked` / `hidden` は非表示

---

## 7. モーション設計

### 7.1 原則

| 項目                 | 値         |
| -------------------- | ---------- |
| 過度なアニメーション | 禁止       |
| duration             | 0.2〜0.3秒 |
| easing               | ease-out   |

### 7.2 使用箇所

- セクションフェードイン
- カード hover
- ページ遷移フェード

> **`prefers-reduced-motion` 対応必須**

---

## 8. ギャラリー設計

### 8.1 一覧表示

- 無限スクロール
- 初期表示: 24件
- 読み込み中: スケルトン表示

### 8.2 作品詳細ページ

1. 写真フル幅表示
2. 横スワイプ対応（複数枚）
3. 下部にタグ表示
4. 最下部にCTA

---

## 9. 信頼セクション設計

### 9.1 Safety（衛生管理）

- 機材・滅菌設備の写真
- 滅菌工程の説明（箇条書き）
- アフターケアの簡潔な説明

### 9.2 価格

- 価格レンジを明示（曖昧にしない）
- 目安表はミニマルなデザイン

---

## 10. 管理画面UI

| 項目                         | 方針                   |
| ---------------------------- | ---------------------- |
| テーマ                       | ダークテーマ           |
| 情報密度                     | 高め                   |
| ナビ                         | 左固定サイドナビ       |
| テーブル余白                 | 少なめ（情報密度重視） |
| 重要操作（削除・非公開など） | 赤系ボタン             |

| -------------- | ------------ |
| コントラスト比 | WCAG AA 以上 |
| タップ領域     | 44px 以上    |
| キーボード操作 | 対応必須     |
| altテキスト    | 全画像に必須 |

---

## 12. レスポンシブ設計

**Mobile First** を基本とする。

### ブレークポイント

```
640px   sm
768px   md
1024px  lg
```

### モバイル時の特則

- 縦1カラム
- CTAは画面幅100%
- ヒーローテキストは短く（改行で調整）

---

## 13. 禁止事項

> [!CAUTION]
> 以下は実装・デザイン時に行ってはならない。

- グラデーション多用
- 影の多用
- アクセント以外の多色展開
- 過度なアニメーション
- 過密レイアウト（余白不足）

---

## 14. 受け入れ基準

| 基準                   | 確認観点                   |
| ---------------------- | -------------------------- |
| 高級感がある           | 余白・タイポ・写真バランス |
| 余白が十分             | セクション間・コンテンツ間 |
| CTAがうるさくない      | 1セクション1CTA以下        |
| 写真が主役             | テキストが写真を圧迫しない |
| モバイルで美しい       | 1カラム・タイポ・余白      |
| スクロールが気持ちいい | フェードイン・適切な密度   |
