# INK ADDICTION LP – テスト設計 v1.0

受領日: 2026-02-27  
ステータス: 確定

---

## 1. 目的

本書は、INK ADDICTION LP の品質を担保するためのテスト観点・テストケース分類・受け入れ基準を定義する。

本プロジェクトにおける最重要テスト対象は以下の3点。

| 優先度 | テスト対象               |
| ------ | ------------------------ |
| 最重要 | 誤公開防止               |
| 最重要 | 予約事故防止             |
| 最重要 | 問い合わせ取りこぼし防止 |

UIだけでなく「状態遷移」と「承認制御」を重点的に検証する。

---

## 2. テスト分類

1. 機能テスト

3. バリデーションテスト
4. 非機能テスト

6. エッジケーステスト

## 3. ギャラリー機能テスト

### 3.1 作品登録

| テストケース      | 期待結果                  |
| ----------------- | ------------------------- |
| 画像1枚以上で保存 | 保存成功                  |
| 画像0枚で保存     | 保存不可                  |
| 分類ジョブ完了後  | `autoGenre` が保存される  |
| 分類ジョブ完了後  | `confidence` が保存される |

### 3.2 公開制御

| テストケース                  | 期待結果                   |
| ----------------------------- | -------------------------- |
| `finalGenre` 未設定で公開操作 | 公開不可（UIで制御）       |
| `is_published = true`         | Publicページに表示される   |
| `is_deleted = true`           | Publicページに表示されない |
| 公開切替時                    | 監査ログが生成される       |

### 3.3 フィルタ

- `genre` フィルタが機能する
- `tags` 検索が機能する
- ページネーションが正しく動作する

## 4. 自動分類テスト

### 4.1 正常系

| テストケース     | 期待結果                          |
| ---------------- | --------------------------------- |
| 和彫画像を投入   | `autoGenre = 和彫`                |
| アニメ画像を投入 | `autoGenre = アニメ`              |
| 分類完了後       | `confidence` が `0.0〜1.0` 範囲内 |

### 4.2 異常系

| テストケース           | 期待結果            |
| ---------------------- | ------------------- |
| 分類API失敗            | `autoGenre = null`  |
| 分類API失敗            | `confidence = null` |
| 上記状態で管理画面表示 | 警告表示される      |

## 5. スクショ解析テスト

### 5.1 正常抽出

| テストケース | 期待結果                               |
| ------------ | -------------------------------------- |
| OCR成功      | `extraction_status = success`          |
| OCR成功      | `extracted_payload` に候補が生成される |
| OCR成功      | `confidence` が付与される              |

### 5.2 低信頼度

| テストケース       | 期待結果                            |
| ------------------ | ----------------------------------- |
| `confidence < 0.6` | 管理画面で警告表示                  |
| 承認前のスロット   | `availability_slots` に登録されない |

### 5.3 承認処理

| テストケース           | 期待結果                          |
| ---------------------- | --------------------------------- |
| 承認ボタン押下         | `status = published` で登録される |
| `draft` 状態のスロット | Public非表示                      |
| 承認操作               | 監査ログが生成される              |

### 5.4 失敗時

| テストケース | 期待結果                      |
| ------------ | ----------------------------- |
| OCR失敗      | `extraction_status = failed`  |
| OCR失敗      | `failure_reason` が保存される |
| OCR失敗後    | 手動入力UIが使用可能          |

## 7. 問い合わせテスト

### 7.1 正常送信

| テストケース       | 期待結果                       |
| ------------------ | ------------------------------ |
| 必須項目入力で送信 | `201` 返却                     |
| 送信後             | DBに `Inquiry` が保存される    |
| 添付ファイルあり   | `inquiry_uploads` に保存される |
| 送信後             | ユーザーへ自動返信メール送信   |
| 送信後             | 管理者へ通知メール送信         |

### 7.2 バリデーション

| テストケース                  | 期待結果      |
| ----------------------------- | ------------- |
| `ageConfirmed = false` で送信 | 拒否（`400`） |
| 不正な `email` 形式           | 拒否（`400`） |
| 添付サイズ超過                | 拒否（`400`） |
| 不正MIME                      | 拒否（`400`） |
| 同一IPから連続送信            | `429` 返却    |

### 7.3 二重送信防止

| テストケース                     | 期待結果         |
| -------------------------------- | ---------------- |
| 同一 `idempotency-key` で2回送信 | 重複保存されない |

## 14. 回帰テスト対象

デプロイ・修正のたびに必ず再確認する項目。

- 作品公開ロジック（`finalGenre` チェック）
- スロット公開ロジック（承認制御）
- `inquiry` 保存
- RLSポリシー
- 監査ログ生成

---

## 15. 受け入れ基準

> [!IMPORTANT]
> 以下を全て満たさない限りリリース不可。

| 基準                    | 内容                     |
| ----------------------- | ------------------------ |
| 未承認データ            | 絶対にPublic表示されない |
| `finalGenre` 未設定作品 | 公開不可                 |
| OCR結果                 | 承認前に公開されない     |
| 問い合わせ              | 必ずDB保存される         |
| 重要操作                | 全て監査ログに残る       |
| モバイル表示            | 視覚崩れなし             |
